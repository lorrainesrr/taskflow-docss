# Guide: From Start to Finish

This guide shows how to use the TaskFlow API in practice, from scratch. Each step uses the result of the previous one, so follow the order.

---

## Step 1: Create a project

Before anything else, you need a project. The `project_id` returned here will be used in the next step.

**Request**

```bash
curl -X POST https://test-writechoice.onrender.com/projects \
  -H "Authorization: Bearer tf-live-4a8f2c1e9b3d7e6a" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "API Documentation",
    "description": "Complete rewrite of the public documentation."
  }'
```

**Response**

```json
{
  "project_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "name": "API Documentation",
  "description": "Complete rewrite of the public documentation.",
  "created_at": "2025-05-12T10:30:00.000Z"
}
```

Save the `project_id`. You will need it in the next step.

---

## Step 2: Create a task

With the `project_id` in hand, create your first task. It is automatically created with the status `pending`.

**Request**

```bash
curl -X POST https://test-writechoice.onrender.com/tasks \
  -H "Authorization: Bearer tf-live-4a8f2c1e9b3d7e6a" \
  -H "Content-Type: application/json" \
  -d '{
    "project_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "title": "Write introduction page",
    "description": "Draft the overview and getting started sections."
  }'
```

**Response**

```json
{
  "task_id": "f9e8d7c6-b5a4-3210-fedc-ba9876543210",
  "project_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "title": "Write introduction page",
  "description": "Draft the overview and getting started sections.",
  "status": "pending",
  "created_at": "2025-05-12T10:35:00.000Z",
  "updated_at": "2025-05-12T10:35:00.000Z"
}
```

Save the `task_id`. You will use it to update or delete this task.

---

## Step 3: List tasks in the project

To see all tasks in a project, pass the `project_id` in the URL.

**Request**

```bash
curl -X GET https://test-writechoice.onrender.com/projects/a1b2c3d4-e5f6-7890-abcd-ef1234567890/tasks \
  -H "Authorization: Bearer tf-live-4a8f2c1e9b3d7e6a"
```

**Response**

```json
{
  "project_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "total": 1,
  "tasks": [
    {
      "task_id": "f9e8d7c6-b5a4-3210-fedc-ba9876543210",
      "project_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
      "title": "Write introduction page",
      "description": "Draft the overview and getting started sections.",
      "status": "pending",
      "created_at": "2025-05-12T10:35:00.000Z",
      "updated_at": "2025-05-12T10:35:00.000Z"
    }
  ]
}
```

---

## Step 4: Update a task

Use the `task_id` to update the status, title, or description of a task. At least one field must be provided.

**Request**

```bash
curl -X PATCH https://test-writechoice.onrender.com/tasks/f9e8d7c6-b5a4-3210-fedc-ba9876543210 \
  -H "Authorization: Bearer tf-live-4a8f2c1e9b3d7e6a" \
  -H "Content-Type: application/json" \
  -d '{
    "status": "in_progress"
  }'
```

**Response**

```json
{
  "task_id": "f9e8d7c6-b5a4-3210-fedc-ba9876543210",
  "project_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "title": "Write introduction page",
  "description": "Draft the overview and getting started sections.",
  "status": "in_progress",
  "created_at": "2025-05-12T10:35:00.000Z",
  "updated_at": "2025-05-12T11:00:00.000Z"
}
```

---

## Step 5: Delete a task

When a task is no longer needed, you can delete it. This action cannot be undone.

**Request**

```bash
curl -X DELETE https://test-writechoice.onrender.com/tasks/f9e8d7c6-b5a4-3210-fedc-ba9876543210 \
  -H "Authorization: Bearer tf-live-4a8f2c1e9b3d7e6a"
```

**Response**

```json
{
  "message": "Task 'f9e8d7c6-b5a4-3210-fedc-ba9876543210' was successfully deleted."
}
```
