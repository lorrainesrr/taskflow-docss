# Create Project

Creates a new project. The `project_id` returned is required to create tasks.

**You must create a project before any other operation.**

---

## Request
```
POST /projects

```
### Headers

| Header | Value |
|---|---|
| `Authorization` | `Bearer tf-live-4a8f2c1e9b3d7e6a` |
| `Content-Type` | `application/json` |

### Body

| Field | Type | Required | Description |
|---|---|---|---|
| `name` | string | Yes | Name of the project |
| `description` | string | No | Optional description of the project |

### Example

```json
{
  "name": "API Documentation",
  "description": "Complete rewrite of the public documentation."
}
```

---

## Responses

### 201 — Project created

```json
{
  "project_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "name": "API Documentation",
  "description": "Complete rewrite of the public documentation.",
  "created_at": "2025-05-12T10:30:00.000Z"
}
```

| Field | Type | Description |
|---|---|---|
| `project_id` | string (uuid) | Unique identifier for the project. Use it in the next endpoints. |
| `name` | string | Name of the project |
| `description` | string or null | Project description, if provided |
| `created_at` | string (date-time) | Date and time of creation |

### 400 — Bad request

Occurs when the `name` field is missing or empty.

```json
{
  "error": "Bad Request",
  "message": "The 'name' field is required and must be a non-empty string."
}
```

### 401 — Unauthorized

Occurs when the API key is missing or invalid.

```json
{
  "error": "Unauthorized",
  "message": "Missing or invalid API key. Provide a valid key in the Authorization header as: Bearer <api_key>"
}
```
